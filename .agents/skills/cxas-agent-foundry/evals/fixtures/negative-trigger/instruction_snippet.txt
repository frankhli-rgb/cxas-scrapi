## Connectivity Routing Taskflow (excerpt)

### Step 2: Identify connectivity type

Trigger: Customer describes a connectivity issue that is NOT home internet.

Action: Route to mobile_troubleshooting taskflow.
  - Call get_device_info to identify the handset
  - Begin mobile diagnostics

### Why this fails

The trigger uses a negative condition: "NOT home internet."
The model interprets this in two ways:

1. CORRECT: "The issue is clearly mobile, skip home internet entirely"
   -> routes directly to mobile_troubleshooting

2. INCORRECT: "I need to verify it's NOT home internet first"
   -> calls check_home_internet to rule it out
   -> then routes to mobile_troubleshooting
   -> eval fails because check_home_internet was not expected

Words like "internet," "connection," and "online" are ambiguous.
The model hedges by checking home internet first when the user
doesn't explicitly say "mobile" or "phone."

Fix: rewrite trigger as an affirmative condition:
  "Customer describes a MOBILE or CELLULAR connectivity issue"
  instead of "NOT home internet." Remove the negation.
